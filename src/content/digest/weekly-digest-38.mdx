### Temporal: 9-летний путь к правильной работе со временем в JavaScript
Работа с датами в JavaScript всегда была головной болью. Стандартный объект `Date` известен своими странностями, а популярные библиотеки вроде [Moment.js](https://momentjs.com/?utm_source=frontend&utm_medium=digest&utm_campaign=shabalin_online) стали временным решением. В 2017 году разработчица Moment.js предложила новый API — [Temporal](https://tc39.es/proposal-temporal/docs/?utm_source=frontend&utm_medium=digest&utm_campaign=shabalin_online), который должен раз и навсегда исправить ситуацию. Сейчас этот стандарт уже почти готов, его [поддерживают всё больше браузеров](https://caniuse.com/temporal?utm_source=frontend&utm_medium=digest&utm_campaign=shabalin_online), хотя Safari и Node.js пока отстают. Статья от Bloomberg подробно рассказывает, как мы к этому пришли.

### Итоги заседания TC39: новые возможности языка
На прошлой неделе прошло заседание комитета TC39, который определяет будущее JavaScript. [Сара Гудинг](https://socket.dev/blog/tc39-advances-temporal-to-stage-4?utm_source=frontend&utm_medium=digest&utm_campaign=shabalin_online) собрала главные новости: предложение **Temporal** продолжает продвигаться, также были рассмотрены **импорт текстовых файлов** (`Import Text`), **доступ к стеку ошибок** (`Error Stack Accessor`) и новый метод для итераторов **`Iterator Includes`**.

## ⚙️ Node.js
### Петиция против использования ИИ‑кода в ядре Node.js
Долгое обсуждение применения AI‑ассистентов в проектах OpenJS Foundation (к которым относится и Node.js) привело к появлению петиции, ограничивающей добавление кода, сгенерированного нейросетями, в ядро Node.js. В сообществе Reddit мнения разделились: большинство поддерживает использование ИИ, но выступает против гигантских пул‑реквестов, которые трудно рецензировать.

### Зачем Node.js нужна виртуальная файловая система
Matteo Collina предложил добавить в ядро Node.js виртуальную файловую систему (VFS), которая позволит импортировать модули, существующие только в памяти. Это пригодится для изоляции кода (песочниц), создания единых исполняемых файлов и тестирования. Уже сейчас можно попробовать [пользовательскую версию VFS](https://github.com/platformatic/vfs?utm_source=frontend&utm_medium=digest&utm_campaign=shabalin_online), хотя она имеет [некоторые ограничения](https://blog.platformatic.dev/why-nodejs-needs-a-virtual-file-system?utm_source=frontend&utm_medium=digest&utm_campaign=shabalin_online#why-vfs-needs-to-live-in-core-nodejs).
🔗 [Подробности в статье](https://blog.platformatic.dev/why-nodejs-needs-a-virtual-file-system?utm_source=frontend&utm_medium=digest&utm_campaign=shabalin_online)
> Примечание: сам PR с реализацией VFS стал одним из поводов для петиции об ИИ‑коде.

### Проблемы и преимущества worker_threads в Node.js
На примере WebSocket‑ориентированного SDK показано, как использование воркеров (worker_threads) помогло избежать «голодания» event loop. Несмотря на то что многопоточность в Node.js имеет свои ограничения по сравнению с другими языками, такой подход успешно справляется с выносом тяжёлых внутренних операций в отдельный поток.
🔗 [Практический пример от Aaron Harper (Inngest)](https://www.inngest.com/blog/node-worker-threads?utm_source=frontend&utm_medium=digest&utm_campaign=shabalin_online)
