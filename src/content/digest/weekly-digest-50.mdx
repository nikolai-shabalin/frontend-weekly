## 🧪 HTML и Платформа

### Safari 27 beta приносит много новых возможностей веб-платформы
Apple показала [что нового появилось в WebKit и Safari 27 beta](https://webkit.org/blog/17967/news-from-wwdc26-webkit-in-safari-27-beta/ "webkit.org"). Релиз большой: кастомизируемый `<select>`, новые CSS-свойства, поддержка `sizes="auto"` для изображений и десятки других улучшений. Для разработчика это важный сигнал: привычные элементы HTML постепенно становятся гибче, а часть интерфейсов, которые раньше требовали тяжелого JavaScript, можно будет собирать ближе к нативной платформе.

### HTML-first подход помог удвоить число завершенных форм
История про [HTML-first сайт, который удвоил число пользователей за ночь](https://mohkohn.co.uk/writing/html-first/ "mohkohn.co.uk"), хорошо показывает, почему базовые формы все еще важны. После двух дорогих неудачных переработок публичный сервис вернули к обычным HTML-формам, сохранению прогресса на сервере и JavaScript только там, где он реально помогает. Чем меньше интерфейс ломается из-за состояния на клиенте, тем проще пользователю дойти до конца.

### Заголовки могут учитывать контекст секции
Manuel Matuzovic разбирает [контекстно-зависимые заголовки в HTML](https://www.matuzo.at/blog/2026/content-aware-headings "www.matuzo.at") и экспериментальный атрибут [`headingoffset`](https://groups.google.com/a/mozilla.org/g/dev-platform/c/rHK-dG4weS0/m/Xgaj7uCQAAAJ). Идея в том, чтобы компонент мог подстраивать уровень заголовка под место, куда его вставили. Это полезно для дизайн-систем: карточка, секция или виджет не должны ломать структуру страницы только потому, что переехали в другую часть макета.

- Google улучшил [Baseline Checker](https://web.dev/blog/baseline-ga-checker-update): теперь проще выбрать подходящую Baseline-цель для сайта без ручного экспорта данных из Analytics.
- В Safari 27 beta переписали [ESM loader ближе к стандартам](https://javascriptweekly.com/link/186272/web), что улучшает поддержку top-level `await`, а WebAssembly получил [поддержку JSPI](https://javascriptweekly.com/link/186273/web) для более удобной работы с промисами между JS и WASM.
- CodePen разрешил превращать работы, созданные в [новом редакторе 2.0](https://blog.codepen.io/docs/pens/deployment/), в [живые опубликованные сайты](https://bsky.app/profile/codepen.io/post/3mnfgn4oxiy2d). Для маленьких демо это сокращает путь от идеи до публичной страницы.

## 🎨 CSS и дизайн

### CSS `@function` делает стили ближе к настоящим вычислениям
В статье про [основы и developer experience CSS `@function`](https://frontendmasters.com/blog/the-fundamentals-and-dev-experience-of-css-function/ "frontendmasters.com") объясняется новая возможность объявлять функции прямо в CSS и возвращать значения, которые потом используются в свойствах. Сейчас [поддержка `@function`](https://caniuse.com/mdn-css_at-rules_function) ограничена Chromium-браузерами, а инструменты еще шероховатые, но направление понятное: часть повторяющейся логики можно будет держать в CSS, не вынося ее в препроцессор или JavaScript.

### CSS gap decorations получают практические примеры
Brecht De Ruyte подробно показывает, как [оформлять промежутки между grid и flex-элементами](https://utilitybend.com/blog/css-is-filling-the-gaps-with-rules-a-way-to-style-gaps-in-grid-and-flex "utilitybend.com"). Раньше линии между карточками часто делали через `border`, псевдоэлементы или дополнительные обертки. Новый подход позволяет стилизовать именно `gap`, поэтому сетка остается чище, а оформление не смешивается с содержимым элементов.

### Переходы между страницами можно делать без JavaScript
Материал про [CSS View Transitions между документами](https://www.rotecodefraktion.de/en/blog/css-view-transitions-zwischen-seiten/ "www.rotecodefraktion.de") показывает, как обычный сайт может получить плавные переходы между страницами всего через CSS. Это не заменяет продуманную навигацию, но помогает сделать многостраничный сайт визуально ближе к приложению без клиентского роутера.

### Grid Lanes получили большой интерактивный справочник
Команда WebKit запустила [The Field Guide to CSS Grid Lanes](https://gridlanes.webkit.org/ "gridlanes.webkit.org") и отдельно [рассказала о запуске справочника](https://webkit.org/blog/18098/introducing-the-field-guide-to-grid-lanes/). Это набор интерактивных примеров про линии и области в CSS Grid. Хороший ресурс, чтобы перестать воспринимать Grid как магическую таблицу и начать осознанно раскладывать сложные блоки.

- В статье про [scrubbable staggered animation на CSS `@function`](https://master.dev/blog/scrubbable-staggered-animation-with-css-function/ "master.dev") показывают, как формулой описать повторяемую задержку для анимаций, прокрутки или других входных значений.
- [CSS Wordle](https://css-questions.com/css-wordle) превращает изучение CSS-возможностей в игру: нужно угадать фичу по подсказкам.
- [FontSelf](https://www.fontself.app/ "www.fontself.app") помогает настроить Google Fonts для self-hosting: выбрать шрифт, скачать пакет и получить готовый CSS.
- В номинациях [Tiny Awards](https://tinyawards.net) собирают маленькие, творческие и handmade-проекты из веба - хороший повод посмотреть на веб не только как на продуктовую платформу.

## ♿ Доступность и UX

### Порядок фокуса обычно не нужно размечать вручную
Eric Bailey объясняет, почему [не стоит отдельно аннотировать focus order](https://ericwbailey.design/published/you-probably-shouldnt-be-annotating-focus-order/ "ericwbailey.design"). Если HTML семантический, а визуальный порядок совпадает с порядком чтения, клавиатурная навигация чаще всего уже работает правильно. Когда приходится рисовать сложную схему фокуса, это может быть признаком, что структура страницы стала слишком запутанной.

### Мифы о доступности лучше разбирать конкретными аргументами
Stéphanie Walter собрала [9 мифов о доступности и ответы на них](https://stephaniewalter.design/blog/9-accessibility-myths-and-pushbacks-and-how-to-answer-them/). Такие возражения часто звучат одинаково: "это дорого", "нашим пользователям не нужно", "мы поправим потом". Статья помогает отвечать не общими лозунгами, а практическими объяснениями про качество продукта, риски и реальные сценарии пользователей.

### Гибридные мобильные приложения требуют осознанного выбора стека
В [гайде по ландшафту hybrid mobile app development](https://piccalil.li/blog/a-front-end-developers-guide-to-the-hybrid-mobile-app-development-landscape/) объясняется, чем отличаются подходы вроде WebView, Capacitor, React Native и PWA. Для фронтенд-разработчика важно понимать компромисс: где вы пишете почти обычный веб, где получаете доступ к нативным API, а где начинаете жить в отдельной мобильной архитектуре.

- Smashing Magazine рассказывает, [как подготовить дизайн-систему к работе с AI](https://www.smashingmagazine.com/2026/06/how-make-design-system-ai-ready/): компонентам нужны ясные правила, токены, описания и ограничения, чтобы модели не собирали интерфейс наугад.
- [Micromodal.js](https://javascriptweekly.com/link/186295/web "micromodal.vercel.app") помогает делать доступные модальные окна по WAI-ARIA-практикам и добавляет удобства поверх нативного [`<dialog>`](https://javascriptweekly.com/link/186296/web): hooks, блокировку прокрутки, анимации и закрытие по фону.

## ⚛️ React и фреймворки

### Angular 22 стабилизирует формы, доступность и асинхронную реактивность
Google выпустил [Angular v22](https://javascriptweekly.com/link/186267/web "blog.angular.dev"). В стабильный или production-ready статус перешли [Signal Forms](https://javascriptweekly.com/link/186268/web), [Angular Aria](https://javascriptweekly.com/link/186269/web) и [асинхронная реактивность](https://javascriptweekly.com/link/186270/web). Роутер также получил экспериментальную работу с браузерным [Navigation API](https://javascriptweekly.com/link/186271/web). Это релиз про то, как Angular подтягивает современные идеи реактивности, не отказываясь от своего полного фреймворк-подхода.

### VoidZero присоединяется к Cloudflare
[VoidZero присоединяется к Cloudflare](https://javascriptweekly.com/link/186265/web "voidzero.dev") - это важная новость для всего мира Vite, Vitest, Rolldown и Oxc. Проекты остаются MIT-лицензированными, а команда продолжит ими руководить. В отдельном [посте команды Vite](https://javascriptweekly.com/link/186266/web) Cloudflare объявила фонд на 1 миллион долларов для мейнтейнеров и контрибьюторов Vite-экосистемы.

### Rust-порт React Compiler готовят к тестированию
[Rust-версия React Compiler](https://javascriptweekly.com/link/186275/web) уже готова к тестированию и близка к слиянию. Компилятор нужен, чтобы React мог лучше оптимизировать компоненты автоматически, а Rust-порт делает инструментальную часть быстрее и удобнее для больших проектов. Для разработчиков это не мгновенная магия в UI, но важный шаг к более зрелой сборке React-приложений.

- [TanStack Table v9 Beta](https://javascriptweekly.com/link/186276/web) обновляет headless-движок таблиц для React, Preact, Solid, Vue, Angular, Svelte и Lit. Headless означает, что библиотека отвечает за состояние и поведение таблицы, а внешний вид вы строите сами.
- [Taiga UI 5.10](https://javascriptweekly.com/link/186306/web) обновляет большой набор компонентов для Angular, где уже больше 50 UI-элементов.
- [Astro Photo Grid](https://github.com/kydecker/astro-photo-grid) предлагает минимальную одностраничную фотогалерею с lightbox для Astro.

## 📦 JavaScript

### Source maps помогают отлаживать продакшен, но требуют контроля
В статье [Everything You Need to Know About Source Maps](https://javascriptweekly.com/link/186289/web) объясняется, как source maps связывают минифицированный код с исходниками. Это удобно, когда нужно понять ошибку из продакшена, но публикация source maps без правил доступа может раскрыть структуру проекта и куски исходного кода.

### Geometric.js добавляет удобную геометрию в JavaScript
[Geometric.js](https://javascriptweekly.com/link/186291/web "www.harryjstevens.com") - библиотека для работы с полигонами, bounding boxes, отражениями, интерполяцией и вращением. В [примерах](https://javascriptweekly.com/link/186292/web) видно, как это применять в графике и визуализациях, а для проверки пересечения фигур есть [отдельная функция](https://javascriptweekly.com/link/186293/web). Это полезно, когда в интерфейсе появляется карта, редактор, диаграмма или любая сцена с координатами.

### NoLoJS собирает альтернативы JavaScript-паттернам на HTML и CSS
[NoLoJS](https://aarontgrogg.github.io/NoLoJS/ "aarontgrogg.github.io") показывает, какие привычные JS-задачи можно решить HTML, CSS или совсем небольшим скриптом. Внутри есть примеры для аккордеонов, модальных окон, ленивой загрузки и других стандартных сценариев. Это не призыв отказаться от JavaScript, а напоминание сначала проверить, не умеет ли платформа это сама.

- [RunJS 4](https://javascriptweekly.com/link/186280/web) обновляет песочницу для быстрого запуска JavaScript-фрагментов.
- [eslint-plugin-functional 10.0](https://javascriptweekly.com/link/186301/web) добавляет правила линтера, которые запрещают мутации и подталкивают код к функциональному стилю.
- [PDFSlick 4.0](https://javascriptweekly.com/link/186297/web "pdfslick.dev") обновляет PDF-viewer для React, Solid, Svelte и vanilla JS-приложений; в [демо](https://javascriptweekly.com/link/186298/web) можно посмотреть режимы просмотра, большие документы и аннотации.
- [Draco.js](https://javascriptweekly.com/link/186304/web) - чистый JavaScript-загрузчик Draco mesh для three.js от mrdoob, с отдельным [демо](https://javascriptweekly.com/link/186305/web).
- [tsParticles 4](https://particles.js.org/ "particles.js.org") обновляет движок частиц для конфетти, огня, снега и похожих эффектов; рядом есть [ribbons](https://ribbons.js.org/) и [живые демо пресетов](https://particles.js.org/playground/presets).

## 📘 TypeScript

### TypeScript умеет выводить типовые переменные, но важно понимать границы
Nicolas Laurent разбирает, [как TypeScript выводит type variables](https://javascriptweekly.com/link/186290/web). Но вывод типов не телепатия: если данные слишком общие или связей между аргументами не хватает, типы нужно помогать уточнять.

- [ts-loader 9.6](https://javascriptweekly.com/link/186302/web) обновляет TypeScript-loader для webpack и [возвращает поддержку webpack 4](https://javascriptweekly.com/link/186303/web), что важно для проектов, которые еще не переехали на более свежую сборку.
- [Kanel 4.0](https://nodeweekly.com/link/186431/web) генерирует TypeScript-типы из живой Postgres-базы, чтобы схемы данных меньше расходились с кодом приложения.
- [`babel-plugin-zod-hoist`](https://javascriptweekly.com/link/186299/web "github.com") от автора [Slonik](https://javascriptweekly.com/link/186300/web) выносит определения Zod-схем выше по файлу и помогает ускорять валидацию.

## ⚙️ Node.js

### npm v12 перестанет запускать install scripts по умолчанию
GitHub рассказал, что [npm v12 больше не будет автоматически выполнять install scripts](https://nodeweekly.com/link/186406/web "github.blog") вроде `preinstall`, `install` и `postinstall`, если проект явно не разрешил их через новый workflow. Подготовиться можно уже сейчас через [npm 11.16.0](https://nodeweekly.com/link/186407/web): он предупреждает, какие пакеты будущая версия заблокировала бы. Это важное изменение для безопасности, потому что install scripts могут запускать произвольный код при установке зависимости.

### Node.js меняет график релизов и нумерацию
Luciano Mammino подробно объясняет [новый релизный цикл и версии Node.js](https://nodeweekly.com/link/186408/web "nodejsdesignpatterns.com"). Начиная с Node 27 проект переходит к одному major-релизу в год и добавляет alpha-канал для тестирования. Для команд это означает более предсказуемое планирование обновлений: проще заранее понять, когда ждать breaking changes, а когда безопаснее оставаться на LTS.

### Supply chain-проблемы в npm продолжают давить на экосистему
На фоне продолжающихся [npm worms и supply chain атак](https://nodeweekly.com/link/186410/web) вспоминают [отчет об уязвимости еще из 2016 года](https://nodeweekly.com/link/186411/web), который предсказывал похожие проблемы. Параллельно [Drizzle ORM](https://nodeweekly.com/link/186412/web) столкнулся с тем, что команда [не может публиковать новые версии в npm](https://nodeweekly.com/link/186413/web) из-за лимита метаданных реестра: у пакета уже [1 397 опубликованных версий](https://nodeweekly.com/link/186414/web). Даже инфраструктурные лимиты внезапно становятся частью надежности зависимостей.

### Node-RED 5.0 сильно обновляет редактор
[Node-RED 5.0](https://nodeweekly.com/link/186421/web "nodered.org") получил самое большое обновление редактора за историю проекта: новые боковые панели, темную тему, паузу debug output и вызов Link nodes из Function nodes. Это low-code среда на Node.js, где логику собирают из блоков, поэтому улучшения редактора напрямую влияют на скорость работы.

- Новые версии [Node.js v26.x, 24.x и 22.x](https://nodeweekly.com/link/186409/web) готовятся закрыть уязвимости, самая серьезная из которых отмечена как HIGH.
- Исследование [ESM provision by popular npm packages](https://nodeweekly.com/link/186416/web "github.com") показывает, что 38% популярных пакетов уже публикуют ESM, а 16% стали ESM-only.
- Из-за бага npm tooling несколько односимвольных пакетов были ошибочно [помечены как security holders](https://nodeweekly.com/link/186415/web), включая `c`, `i`, `n` и `x`; для реестра это редкий, но показательный сбой метаданных.
- [Playwright fixtures API](https://nodeweekly.com/link/186418/web) разобрали изнутри: статья показывает, как Playwright понимает нужные фикстуры по сигнатуре тестовой функции.
- [pnpm 11.5](https://javascriptweekly.com/link/186279/web) обновляет пакетный менеджер на фоне общей работы экосистемы над безопасностью установок.
- [Bonsai](https://nodeweekly.com/link/186422/web "danfry1.github.io") предлагает безопасный язык выражений для пользовательских правил, фильтров и шаблонов без `eval`; для экспериментов есть [playground](https://nodeweekly.com/link/186423/web).
- [DepsGuard](https://nodeweekly.com/link/186424/web "depsguard.com") проверяет и ужесточает настройки npm, pnpm, Yarn и Bun: отключает install scripts, включает cooldowns и следит за provenance.
- [Commander.js 15.0](https://nodeweekly.com/link/186425/web) стал ESM-only, а версия 14 будет поддерживаться до мая 2027 года для проектов, которым нужно время на миграцию.
- [NodeAV 6.0](https://nodeweekly.com/link/186426/web) дает прямой доступ к C API FFmpeg для работы с кодеками, фильтрами и потоками.
- [Juice 12.1](https://github.com/Automattic/juice) встраивает CSS прямо в HTML, что часто нужно при сборке email-шаблонов; библиотеку используют и в инструментах вроде [MJML и node-email-templates](https://nodeweekly.com/link/186429/web).
- [Cucumber.js 13.0](https://nodeweekly.com/link/186430/web) перестроил параллельный runtime тестов на worker threads.
